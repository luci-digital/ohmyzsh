# luci-devops

Production-oriented, reproducible **LuciDigital DevOps platform** — a sovereign
substrate unifying shell, web, SCM, orchestration, and infra under **Nix flakes**
and **Podman** (no Docker). LDS 800.000 @ 741 Hz · Genesis Bond: ACTIVE.

## Layout

```text
flake.nix              Reproducible dev shells + packages + checks (Nix flakes)
modules/
  shell/               LuciVerse zsh environment (LuciDigital customizations)
  web/luci-frontend/   TanStack Start + Vite + Tailwind frontend
  scm/                 Source control: gogs (Gitoxide image), luci-vcs (Rust crate)
  orchestration/       Podman compose stack: caddy, coder, ipfs, homestar, ray, …
  infra/               dns (Quad9), ipv6 schema, foundationdb, postgres, secrets
  docs/                specs (DID handles, open-compute), security, architecture
  ci/                  github-actions (canonical at .github/), Nix checks
  legacy/              Upstream Oh My Zsh, relocated verbatim (nothing deleted)
```

Functional GitHub config (`.github/`), editor config (`.vscode/`,
`.editorconfig`), governance (`LICENSE`, `CONTRIBUTING`, `SECURITY`,
`CODE_OF_CONDUCT`), agent guide (`AGENTS.md`), and the devcontainer
(`.devcontainer/`) stay at the repo root where their tooling expects them.

## Quick start (Nix)

```sh
nix develop              # full platform shell (rust + node + podman + caddy + zsh)
nix develop .#web        # frontend only          → modules/web/luci-frontend
nix develop .#scm        # luci-vcs (Rust)        → modules/scm/luci-vcs
nix develop .#orchestration
nix build .#luci-vcs     # build the VCS substrate
nix flake check          # run luci-vcs test suite
nix fmt                  # format Nix + shell + Rust in place
```

### Format / check apps (`nix run .#<app>`)

| App | Runs |
|---|---|
| `web-install` | `pnpm install --frozen-lockfile` (web) |
| `web-build` | `pnpm build` (web) |
| `web-test` | `pnpm test` (web) |
| `cargo-check` | `cargo check` (luci-vcs) |
| `cargo-test` | `cargo test` (luci-vcs) |
| `shellcheck` | shellcheck our `*.sh` (legacy OMZ excluded) |
| `shfmt` | `shfmt -d` our `*.sh` |
| `zsh-syntax` | `zsh -n` over OMZ + shell module |
| `compose-config` | validate the Podman compose file |
| `ci` | full local gate: cargo + web + shell + compose |

> The flake's `flake.lock` is generated by `nix flake lock` on first use.
> Nix and Podman are **not** vendored in this repo; the flake declares them.

## Deploy (Podman)

```sh
podman-compose -f modules/orchestration/podman/podman-compose.yml up -d
```

The compose stack wires Gogs+Gitoxide SCM, the Lucia orchestrator, Caddy ingress
(IPv6-native, `2602:F674::/40`), Coder CDE, OpenDeepWiki, Ray, IPFS/Kubo, and
Homestar/IPVM. Build contexts reference the `scm/gogs` module; ingress mounts
`../caddy/Caddyfile`.
