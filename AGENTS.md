### Deployment

The app uses `@tanstack/react-start` which targets Node.js by default. To deploy:

- **Node.js / Docker**: `pnpm build` then serve `.output/` with `node .output/server/index.mjs`
- **Cloudflare Workers**: load skill `@tanstack/start-client-core#start-core/deployment` for adapter steps
- **Vercel / Netlify**: same skill covers adapter configuration
- No platform-specific lock-in in the base scaffold; portable by default

---

### Key Architectural Decisions

- **File-based routing**: routes live in `src/routes/`. The router plugin auto-generates `src/routeTree.gen.ts` — do not edit this file manually.
- **`getRouter()` factory** in `src/router.tsx`: single source of truth for router config; imported by both client and server entry points.
- **Root route shell** (`src/routes/__root.tsx`): renders the full HTML document via `shellComponent`. `HeadContent` and `Scripts` must be present.
- **Theme persistence**: inline `THEME_INIT_SCRIPT` in `<head>` prevents flash-of-wrong-theme on hydration. Theme state is stored in `localStorage` under the key `theme`.
- **Path aliases**: `#/*` and `@/*` both resolve to `src/`. Prefer `#/` for project imports (matches `package.json#imports`).

---

### Known Gotchas

- `src/routeTree.gen.ts` is auto-generated by the Vite plugin at dev/build time. It will not exist until you run `pnpm dev` or `pnpm build` for the first time.
- `noUnusedLocals` and `noUnusedParameters` are enabled — remove unused imports/params before committing.
- The scaffold was created with npm (CLI default) and the `node_modules` from that run are in `/tmp/`. Run `pnpm install` in this directory to get pnpm's lockfile and `node_modules`.

### Compliance Service

ISO compliance is monitored continuously by `core/iso_compliance.lua` on the Lua side, surfaced via the web at `/compliance`. The service:

- Runs a full audit every hour (tick via `ngx.timer.every`)
- Covers 8 standards: ISO-27001, 27018, 20022, 23894, 9001, IEC-23053, IEC-22989, IEC-24029
- Emits `compliance_drift` signals to `luci:signal:broadcast` on violations
- Persists audit records to FoundationDB at `/luciverse/compliance/audits/`
- Is triggered from the web via `POST /validate` with `type: "iso_compliance_audit"`
- Agents responsible: Judge Luci (963 Hz, governance), Claude-Veritas (432 Hz, verification)

See [`docs/lua/iso_compliance.lua`](docs/lua/iso_compliance.lua) for the full Lua module spec and wiring instructions.
